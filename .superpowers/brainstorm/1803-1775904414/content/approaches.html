Comment le Studio affiche-t-il la prévisualisation ?

3 façons de connecter le formulaire à l'aperçu — ma recommandation est marquée ⭐

A

Blob URL dynamique ⭐ Recommandé

Studio crée un Blob URL combinant config.js + template/ et l'injecte dans l'iframe. Zéro serveur, fonctionne 100% en local.

config.js (drag & drop ou formulaire)
↓ génère du JS en mémoire
Blob URL = config + template inchangé
↓ chargé dans iframe
Rendu exact du vrai site ✅

Avantages

  • Le template n'est jamais modifié
  • Fonctionne sans serveur local
  • Preview = rendu 100% fidèle
  • Simple à implémenter

Inconvénients

  • Médias (vidéo, photos) non chargés en preview locale — aperçu texte seulement
B

Serveur local (Node/Python)

Un mini serveur tourne en local. Studio écrit config.js dans template/, l'iframe charge localhost:8080. Médias inclus.

Formulaire Studio modifié
↓ écrit config.js sur disque
Serveur local sert template/
↓ iframe = localhost:8080
Rendu complet avec médias ✅

Avantages

  • Médias (vidéo, photos) visibles
  • Rendu complet

Inconvénients

  • Nécessite Node.js ou Python installé
  • Commande à lancer avant d'ouvrir le Studio
  • Écrit dans les fichiers template (risque d'erreur)
C

Prévisualisation inline (sans iframe)

Le Studio reconstruit visuellement les sections clés (Hero, Programme, Lieux…) directement dans sa propre interface, sans charger le vrai template.

Formulaire Studio modifié
↓ JS re-render en temps réel
Sections reconstituées dans le Studio
↓ aperçu "stylisé"
Approche légère mais découplée ⚠️

Avantages

  • Aucune dépendance
  • Très rapide à mettre à jour

Inconvénients

  • Pas le vrai rendu — risque de surprise au déploiement
  • Double maintenance (Studio + template)
  • Ne vaut pas la peine
